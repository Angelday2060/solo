# solo 需求文档

## 1. 产品定义

| 项 | 内容 |
|----|------|
| 名称 | **solo** |
| 形态 | 桌面端应用（技术实现见《Electron-Vue3-NaiveUI-开发说明》） |
| 定位 | 个人事务的统一记录与检索入口，数据归属用户本地或用户明确授权的目标 |
| 不在范围（首期） | 多人协作、企业场景、服务端社交功能 |

## 2. 目标用户与场景

- **用户**：单用户；希望在一个工具内管理日记、开支、笔记、日程，减少应用切换与数据分散。
- **典型场景**：每日随手记、月底对账、碎片知识归档、计划与提醒。

## 3. 功能域概览

| 域 | 名称 | 核心价值 |
|----|------|----------|
| A | 日记 | 按日/时间轴沉淀情绪与事件，可检索 |
| B | 记账 | 收支分类、账户/周期汇总，辅助消费回顾 |
| C | 笔记 | 非日记类长文本与结构化摘录，支持标签与链接 |
| D | 日程 | 事项、截止日期、重复规则、与日历视图配合 |

四域数据在应用内应可通过统一搜索或时间线交叉浏览（首期可实现「分模块搜索」，跨模块全局检索作为增强项）。

## 4. 详细需求

### 4.1 日记

- **记录**：富文本或 Markdown（实现阶段二选一或并存）；自动记录创建/修改时间；可选天气、心情等元数据（字段可配置开关）。
- **组织**：按日期聚合列表；支持自定义标题或「无标题日记录」。
- **检索**：按日期范围、关键词、标签（若支持）过滤。

### 4.2 记账

- **流水**：金额、币种、收支方向、分类（可编辑分类树或扁平列表）、账户（现金/银行卡/电子钱包等）、备注、发生日期。
- **统计**：按日/周/月/年汇总；按分类占比展示（首期可用列表+简单图表占位）。
- **约束**：删除与批量编辑需可撤销或二次确认（防误触）；金额精度与币种规则在实现层固定策略并写入说明。

### 4.3 笔记

- **记录**：支持多笔记；标题、正文、附件占位（首期可仅文件路径或后期再做内嵌预览）。
- **组织**：文件夹或笔记本层级 + 标签；支持置顶、最近编辑排序。
- **关联**：可选「链接到某条日记/某笔账目」作为扩展字段，降低首期可不实现。

### 4.4 日程

- **事项**：标题、详情、开始/截止时间、全天开关、提醒时间（依赖系统通知能力）。
- **重复**：无 / 按天 / 按周 / 按月 / 按年（首期至少支持「无 + 按周」可后续迭代补全）。
- **视图**：列表视图必选；月历/周历为增强项。

### 4.5 跨域与系统级

- **统一入口**：侧边栏或顶栏切换四大模块；全局快捷键打开「快速记录」（实现阶段再定默认落到哪一模块）。
- **数据**：默认本地存储；导出（如 JSON / CSV 账单）与备份路径在后续迭代单独立项。
- **设置**：外观（浅/深色）、数据目录位置（若 Electron 可配置）；界面文案固定为简体中文，不提供语言切换。

## 5. 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 常规数据量下（_notes / 日记数千级、账目万级）列表与搜索无明显卡顿；大数据量需分页或虚拟列表 |
| 可靠 | 写入失败可感知；关键操作有持久化策略说明（单文件 DB / 多文件等由设计文档定义） |
| 隐私 | 无未经说明的远程上报；若未来有云同步须在单独需求中定义加密与认证 |
| 可用性 | 键盘可操作主要路径；空状态有明确引导文案 |

## 6. 首期交付边界（MVP 建议）

- 四大模块均具备：**增删改查**、列表与基础元数据。
- 日记、笔记：关键词检索。
- 记账：分类 + 简单周期汇总。
- 日程：列表 + 单项提醒（若系统通知不可靠则降级为应用内红点/横幅）。
- **不强制**：云同步、OCR、复杂报表、移动端。
