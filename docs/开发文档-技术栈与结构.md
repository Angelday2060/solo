# solo 简要开发文档（Rust + Tauri）

桌面端单机应用：**Rust 宿主（系统 WebView）+ Vue 3 前端**；系统能力（文件路径、SQLite 等）放在 **Rust 侧**，前端通过 **`invoke` 命令 / 事件** 调用。

## 技术栈分层

| 层级 | 技术 |
|------|------|
| 桌面宿主 / 系统集成 | Tauri（Rust `src-tauri` + 各 OS WebView，如 Windows WebView2） |
| Rust 运行时与构建 | Rust（`cargo`）、`tauri-build`、`tauri-codegen` |
| 应用能力与权限 | `src-tauri/capabilities` + `tauri.conf.json`（按 Tauri 版本对应字段命名） |
| 前端开发与生产构建 | Vite、`@vitejs/plugin-vue`、`vue-tsc` |
| 前端与壳通信 | `@tauri-apps/api`（`invoke`、`listen` 等） |
| 前端框架 | Vue 3 |
| 路由 | vue-router |
| UI 组件与主题 | Naive UI、`src/theme` 定制 |
| 图表 | Chart.js |
| 语言 | TypeScript（前端）；Rust（后端命令与业务集成） |
| 本地持久化（推荐落点） | SQLite：`rusqlite`（`bundled` 特性打包 libsqlite，减少系统依赖）；不建议把业务库继续在渲染进程 WASM 做主存储 |
| 共享契约 | TypeScript：`shared/`、`@`、`@shared`；与 Rust `serde` 结构对齐（字段名一致便于 `invoke` 传参序列化） |

## 常用命令（根目录，与 `create-tauri-app` / 官方脚手架一致）

具体脚本名以 `package.json` 为准，常见等价关系如下：

- `npm run tauri dev` — 本地开发（Vite 前端 + Tauri dev）
- `npm run tauri build` — 生产构建（前端 `vite build` + Tauri 打包安装包）

（若仍需纯浏览器预览：`npm run preview` 仅用于不含壳的静态产物，不作为桌面集成测试。）

## 项目目录树（Tauri + Vue 约定）

不含 `node_modules`、`src-tauri/target`、前端 `dist` 及安装包产出目录。

```
solo/
├── docs/                          # 说明文档（含本文件）
├── src-tauri/                     # Rust 宿主：窗口、插件、SQLite、invoke 命令
│   ├── src/
│   │   └── main.rs                # 入口；注册命令、初始化 DB 等
│   ├── build.rs
│   ├── Cargo.toml
│   ├── tauri.conf.json            # 窗口、打包、权限与 security 相关配置
│   ├── capabilities/              # （Tauri 2）能力粒度声明（若使用）
│   └── icons/
├── public/                        # Vite 静态资源
├── shared/                        # 与前端、`invoke` 载荷对齐的 TS 类型
│   └── ledger-types.ts
├── src/                           # Vue 前端（渲染层）
│   ├── assets/
│   ├── router/
│   ├── services/                  # 封装 @tauri-apps/api `invoke`
│   ├── theme/
│   ├── utils/
│   ├── views/
│   ├── App.vue
│   ├── main.ts
│   ├── style.css
│   └── vite-env.d.ts
├── 原型图/                        # 静态 HTML 原型（不参与构建）
├── index.html
├── package.json
├── package-lock.json
├── tsconfig.json
├── tsconfig.node.json
└── vite.config.js                 # 与官方「Vite | Tauri 2」一致：CLI 钩子 + 本文件，无 @tauri-apps/plugin-vite
```

从 Electron 迁出后：**删除或废弃 `electron/` 目录**，账本与 DB 逻辑迁移到 **`src-tauri/src`**（命令函数 + 模块拆分），前端 **`src/services`** 只做 `invoke` 与 UI 适配。
