# 记账模块 — 实体类设计

依据《solo-需求文档》4.2 节与产品原型中的字段约定。与前端 TypeScript、Rust `serde`、SQLite 表结构应对齐字段命名（建议使用 `snake_case` 落库、`camelCase` 仅存 TS 层时在边界转换）。**不区分支付方式/资金账户**，流水无账户类字段。

---

## 1. 枚举与代码表（非独立业务表时可内联）

### 1.1 币种代码 `CurrencyCode`

| 取值 | 含义 | 说明 |
|------|------|------|
| `cny` | 人民币 | 首期与原型一致 |
| `jpy` | 日元 | 首期与原型一致 |

扩展币种时在本表与校验逻辑中同步追加。

### 1.2 收支方向 `LedgerDirection`

| 取值 | 含义 |
|------|------|
| `in` | 收入 |
| `out` | 支出 |

---

## 2. 分类 `LedgerCategory`

分类支持扁平列表；若实现「树」则在同一实体上通过 `parent_id` 表达。

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `id` | string | 是 | 主键，稳定不可变（如 UUID 或带前缀的自定义 id） |
| `name` | string | 是 | 展示名称，用户可编辑 |
| `parent_id` | string \| null | 否 | 父分类 id；`null` 表示根节点；扁平列表可不使用，恒为 `null` |
| `sort_order` | number | 否 | 同级排序，数值越小越靠前 |
| `created_at` | string | 否 | ISO 8601，创建时间 |
| `updated_at` | string | 否 | ISO 8601，最后修改时间 |

---

## 3. 流水（账目）`LedgerTransaction`

核心业务实体，对应「单笔收支记录」。

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `id` | string | 是 | 主键 |
| `amount` | string | 是 | 以字符串存储十进制金额，避免浮点误差；显示与计算按固定小数策略解析 |
| `currency` | `CurrencyCode` | 是 | 该笔币种 |
| `direction` | `LedgerDirection` | 是 | 收入或支出 |
| `category_id` | string \| null | 否 | 关联 `LedgerCategory.id`；空表示未分类 |
| `occurred_on` | string | 是 | 发生日期，`YYYY-MM-DD` |
| `note` | string | 否 | 备注 |
| `created_at` | string | 否 | ISO 8601，记录创建时间 |
| `updated_at` | string | 否 | ISO 8601，最后修改时间 |

**金额精度（实现约定）**：与需求文档一致，在存储层统一小数位数与舍入规则（例如保留两位小数、银行家舍入等），并在《开发说明》中单独立条，本文只定义字段语义。

---

## 4. 实体关系（提要）

- `LedgerTransaction.category_id` → `LedgerCategory.id`（多对一，可空）
- `LedgerCategory.parent_id` → `LedgerCategory.id`（自引用，可选）

统计视图（按日/周/月/年、按分类占比）由查询聚合产生，不作为独立持久化实体类。
