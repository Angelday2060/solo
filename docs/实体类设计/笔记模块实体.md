# 笔记模块 — 实体类设计

依据《solo-需求文档》4.3 节与产品原型中的「文件夹 / 笔记本 · 笔记列表 · 标题 / 正文 / 标签 / 附件占位」约定。与前端 TypeScript、Rust `serde`、SQLite 表结构应对齐字段命名（建议使用 `snake_case` 落库、`camelCase` 仅存 TS 层时在边界转换）。

**与日记的边界**：日记按**日历日**聚合、偏情绪与事件记录；笔记面向**非日期的长文本与结构化摘录**，可归属笔记本、打标签、挂附件，并可选链接到其他模块记录。二者正文首期均建议 **UTF-8 纯文本**；若日后笔记单独支持 Markdown/富文本，需另增格式字段并做数据迁移（日记模块已明确仅纯文本，不在此重复）。

---

## 1. 正文约定

- `body` 存 **UTF-8 纯文本**：不按 Markdown/HTML 解析；入库与检索均为用户输入的字面文本。
- 换行使用实际换行符（如 `\n`）；检索与展示按字面文本处理。
- 界面侧使用多行文本编辑器即可；「结构化摘录」首期由用户自行排版，不另建块级子实体。

---

## 2. 笔记本（文件夹）`NoteNotebook`

组织容器，对应侧栏「文件夹 / 笔记本」层级树。通过 `parent_id` 表达树形结构（与 `LedgerCategory` 自引用模式一致）。

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `id` | string | 是 | 主键，稳定不可变（如 UUID） |
| `name` | string | 是 | 展示名称，用户可编辑 |
| `parent_id` | string \| null | 否 | 父笔记本 id；`null` 表示根级节点 |
| `sort_order` | number | 否 | **同级**排序，数值越小越靠前 |
| `is_pinned` | boolean | 否 | 是否置顶于同级树节点；默认 `false` |
| `created_at` | string | 否 | ISO 8601，创建时间 |
| `updated_at` | string | 否 | ISO 8601，最后修改时间 |

**删除策略（实现约定）**：删除笔记本时，子节点与所含笔记的处理策略（级联删除 / 上移到父级 / 移入「未分类」）在《开发说明》中单独立条；本文仅定义实体字段。

---

## 3. 笔记条目 `Note`

核心业务实体，对应「某笔记本下的一篇笔记」（未归属时可置于 `notebook_id = null` 的「未分类」列表）。

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `id` | string | 是 | 主键，稳定不可变（如 UUID） |
| `notebook_id` | string \| null | 否 | 所属 `NoteNotebook.id`；`null` 表示未分类 / 收件箱 |
| `title` | string \| null | 否 | 标题；`null` 或空串按产品文案展示为「无标题」 |
| `body` | string | 否 | **纯文本**正文；首期允许为空草稿 |
| `is_pinned` | boolean | 否 | 是否置顶；列表默认 **置顶优先**，同组内按 `updated_at` 降序（最近编辑） |
| `sort_order` | number | 否 | **同一 `notebook_id` 内**手动排序；与置顶、最近编辑并存时，实现层约定优先级并在《开发说明》说明 |
| `created_at` | string | 否 | ISO 8601，首次保存时间 |
| `updated_at` | string | 否 | ISO 8601，最后修改时间；驱动「最近编辑」列表排序 |

**正文体积**：超长文本建议在实现层约定单行上限或改用外链存储策略，并在《开发说明》中单独立条；本文仅定义字段语义。

**列表默认排序（查询约定，非持久化字段）**：`is_pinned DESC` → `updated_at DESC` → `sort_order ASC` → `created_at DESC`（具体次序以实现为准，需与原型「置顶 · 最近编辑」一致）。

---

## 4. 标签 `NoteTag` / 关联

需求 4.3 要求「标签」组织与过滤。与日记模块标签模型对齐，采用独立标签表 + 多对多关联。

### 4.1 标签定义 `NoteTag`

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `id` | string | 是 | 主键 |
| `name` | string | 是 | 展示名，唯一（或与规范化名唯一） |
| `created_at` | string | 否 | ISO 8601 |

### 4.2 笔记—标签关联 `NoteTagLink`

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `note_id` | string | 是 | `Note.id` |
| `tag_id` | string | 是 | `NoteTag.id` |

主键可为 `(note_id, tag_id)`。

**MVP 边界**：标签可为首期可选能力；若未实现 UI，仍建议预留表结构以便后续增量启用，避免笔记表反复迁移。

---

## 5. 附件占位 `NoteAttachment`

需求 4.3「附件占位（首期可仅文件路径或后期再做内嵌预览）」。一条笔记可挂多个附件元数据，不内嵌文件二进制。

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `id` | string | 是 | 主键 |
| `note_id` | string | 是 | `Note.id` |
| `file_path` | string | 是 | 本地绝对或相对于数据目录的路径；首期仅存路径，不做内嵌预览 |
| `display_name` | string \| null | 否 | 展示用文件名；缺省可由 `file_path` 推导 |
| `sort_order` | number | 否 | 同一笔记内附件排序 |
| `created_at` | string | 否 | ISO 8601，添加时间 |

**MVP 边界**：可仅落库 + 列表展示路径，打开/预览行为在实现阶段再定。

---

## 6. 跨模块链接（可选扩展）`NoteExternalLink`

需求 4.3「可选链接到某条日记 / 某笔账目」；**首期可不实现**。若实现，建议独立关联表而非在 `Note` 上堆多个可空外键，便于一对多链接与类型扩展。

| 属性 | 类型 | 必填 | 约束 / 说明 |
|------|------|------|-------------|
| `id` | string | 是 | 主键 |
| `note_id` | string | 是 | `Note.id` |
| `target_type` | string | 是 | 链接目标类型，见下表 `NoteLinkTargetType` |
| `target_id` | string | 是 | 目标记录主键（如 `DiaryEntry.id`、`LedgerTransaction.id`） |
| `label` | string \| null | 否 | 可选展示文案；缺省由目标模块查询填充 |
| `created_at` | string | 否 | ISO 8601 |

### 6.1 链接目标类型 `NoteLinkTargetType`

| 取值 | 含义 |
|------|------|
| `diary_entry` | 日记条目 `DiaryEntry` |
| `ledger_transaction` | 账目流水 `LedgerTransaction` |

扩展其他模块时在枚举与校验逻辑中同步追加。目标记录被删除时，链接行处理策略（级联删除 / 保留并标记失效）在《开发说明》约定。

---

## 7. 实体关系（提要）

```
NoteNotebook ──parent_id──► NoteNotebook（自引用树）
Note.notebook_id ──────────► NoteNotebook（多对一，可空）
Note ◄──► NoteTag（多对多，经 NoteTagLink）
Note ◄── NoteAttachment（一对多）
Note ◄── NoteExternalLink（一对多，可选扩展）
NoteExternalLink.target_id ──► DiaryEntry / LedgerTransaction（逻辑外键，跨模块）
```

- **必选（MVP 核心）**：`NoteNotebook`、`Note`。
- **MVP 建议**：`NoteTag` + `NoteTagLink`（与需求「标签」一致；UI 可分期）。
- **MVP 占位**：`NoteAttachment`（路径元数据）。
- **可选扩展**：`NoteExternalLink`。
- **非实体**：关键词检索、按笔记本过滤、置顶/最近编辑排序均由查询产生；编辑器草稿态可不落库直至首次保存。

| 实体 | 建议表名 |

| `NoteNotebook` | `note_notebook` |
| `Note` | `note` |
| `NoteTag` | `note_tag` |
| `NoteTagLink` | `note_tag_link` |
| `NoteAttachment` | `note_attachment` |
| `NoteExternalLink` | `note_external_link` |

建议索引（实现参考）：`note (notebook_id)`、`note (updated_at)`、`note_attachment (note_id)`、`note_tag_link (tag_id)`、`note_external_link (note_id, target_type, target_id)`。
